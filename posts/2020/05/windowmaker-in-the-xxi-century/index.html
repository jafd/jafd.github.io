WindowMaker in the XXI Century

Date [ 2020-05-20 ] Tags [ programming ]

I’m running WindowMaker as my desktop environment. Usually. That is, I keep returning to it.

For the last few years, using Linux meant using Gnome 3. It’s not bad as such, except that it uses far too many resources for things I think don’t have to use that many resources.

Also, it keeps losing functionality I like. One such is moving windows by dragging anywhere on them while Alt is pressed. On Wayland, you cannot restart the desktop shell without losing all the clients. And the shell leaking memory is quite a reason to restart it once in a while.

However, the world had moved on in the last 5 years, and your average Linux/FreeDesktop world grew quite a couple conveniences you don’t want to give up when moving to a simpler window manager. Turns out, WindowMaker is quite inept in some cases, although it’s solvable.

It would be nice, of course, to make the solutions reusable and packageable, so you don’t need to reapply them all.

My experience is about making it all work on Fedora. YMMV on another distro, but as long as it’s mainstream enough, it all should work.
